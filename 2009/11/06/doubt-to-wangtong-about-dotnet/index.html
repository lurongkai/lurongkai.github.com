对10月4日一节课上老师部分话的质疑

By Lu Rongkai

Published 2009-11-06

Contents

ASP.NET是看不到源代码的

质疑：事实上这句话不应该现在说，至少应该说成为“部分开源”，因为ASP.NET依赖的.NET Framework框架在08年1月底时(具体时间是美国时间2008年1月16日)，在微软的MS-RL协议()下公开了其.NET Framework的第一批代码，到现在为止，所有源码都可以从下载到，不完全与开源相同，MS-RL协议只允许开发人员查看源代码而不能修改后发布。但是可以说.NET Framework的确是可以完完整整的看到源代码而并不是王老师所说的“看不到源代码”。另外，用.NET语言(C++、C#、VB.NET、J#、F#、IronPython、IronRuby等基于MSIL和CLR的语言)开发的应用程序，除非经过特殊的“混淆处理”，不然的话可以用.NET Reflector这个工具看到其所有的源代码，不信的话大家可以下载“飞信”这个用.NET语言开发的应用程序，然后用.NET Reflector这个工具载入就可以看到它的源代码了，这一点是其他语言做不到的，因为如果要“看”源代码，那么在编程方面.NET并不是像传说中的那么吝啬，例如在ASP.NET下很著名的MVC框架就是完全开放源代码的，在软件的CodePlex网站上可以找到很多的开源项目。

ASP.NET的安装(运行环境)通常要两三个G，而Java虚拟机才100多M

质疑：老师所说的是Visual Studio吧？如果说ASP.NET的安装(这个说法并不妥当，事实上说ASP.NET与J2EE或JEE的安装都不是一个准确的说法)，那么结合前面所说的，ASP.NET的环境只要提供对应.NET Framework版本支持，附加个Web Server(通常情况下是IIS，但是……下面一条有更为详细的解释)就能跑起来，所以句话对比的地方就成了.NET Framework的安装大小与Java虚拟机的安装大小。不行不承认，Java虚拟机的配置的确是“简单而绿色”，大小一目了然，然后.NET Framework的大小真的很大吗？微软的主页上提供了.NET Framework 3.5的完整下载包()，我们可以发现其大小为

那么如此说来，.NET Framework的确是比Java虚拟机大一倍了，而且需要安装，但是并不是如题所说的需要两三个G，那是开发环境，并且是一个集成开发环境，就是说不单是开发ASP.NET应用程序的环境，而事实上如果只做ASP.NET的开发的话，微软提供了单独的免费的开发组件叫Visual Web Developer，而Visual Studio里的ASP.NET开发时用到的正是它，况且和Java类似，Java虚拟机只有100多M，而如果.NET用记事本+.NET Framework SDK来做的话确实也不是很大，因为只是单纯的编译环境而已。而且，如果说大小的话，也仅是2G多一点，下图是我Microsoft Visual Studio 2008(即9.0)与Microsoft Visual Studio 2010 Beta2(即10.0)的完整安装所占用的硬盘大小(没有安装附加的Microsoft SQL Server Express)：

那么大一倍定就有大一倍的道理，我想从效率上考虑一下，而这部分可以和大家一起来讨论，因为我不太知道Java的运行原理，但是从一些文档可以知道的是，.NET的底层是先将语言编译成为MSIL(微软中间语言)，当然，这语言的类型系统是和CLR匹配的语言，这个就是.NET程序发行时的最终代码，这有点类似于Java的JVM可执行代码(Java字节码)，这也就是Java为什么能跨平台的原因，因为在不同的平台上只要安装对应的JVM(就是Java虚拟机)，那么这些相同的Java字节码就会解释成为不同的平台专门的代码。到些为止似乎.NET也借鉴了Java的技术，然而接下来要讨论的东西就是这样，Java在运行时是由JVM动态的解释成为机器语言，其中有一个解释时的效率损耗，而.NET呢？类似于JVM，JIT(Just In Time)，它会在第一次运行时将MSIL动态的编译成为本平台下的机器码，然而与Java不同的是，这是编译而不是解释，换句话讲，当第二次运行时已经是机器码而非MSIL了，这样就终身的省去了解释的时间，所以说，.NET应用程序的运行时效率不一定比Java低，为什么说不一定呢？因为这需要讨论，Java的发展也很快，毕竟在语言里它的普及率还是要比.NET高的。

IIS只能解析ASP(ASP.NET)的东西

这个似乎不需要太多的讨论，因为IIS已经可以完美的支持PHP了，博客园的老赵做了很详细的解释，具体的报导可以看下这里，而且，IIS跑PHP的性能还不错，比LAMP架构下的Apache要快上一百倍(性能评测报告看下这里)。另外，在开源项目Mono的大力发展下，.NET的应用程序已经可以顺利的在Linux/Unix/Mac下运行了，Mono是Linux/Unix/Mac平台下的一个开源项目，目前该项目已经可以正常的运行.NET下的ASP.NET、Winforms、GTK#程序，其对应的集成开发环境MonoDevelop也相当的完善了。
